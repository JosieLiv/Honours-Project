Injection

Injection vulnerabilities occur when an application improperly validates or filters user
input before it is processed, allowing an attacher to 'inject' malicious code. Once the
backend executes this code, the system can be compromised and data can be leaked. SQL Injection
(SQLi) is when malicious user input is inserted directly into SQL queries resulting in unauthorised
access to a database.
In this lab, you will have access to the web application for Book Nook, where users can browse, buy
and wishlist books. However, the site has been deployed too early and the developers were very sloppy.
Unsanitised input can be passed directly into a database query.
