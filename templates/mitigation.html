Broken Access Control

Strong access control can be implemented using a defense-in-depth approach. This is when mutlipe access
control mechanisms are layered to reduce the risk of a security breach. Even if one layer fails,
there will still be protection in place. Here is a list of mitigation strategies:

    Server-side access control as client-side validation checks are not enough
    Principle of Least Privilege, meaning all users are granted only the minimum permissions needed
    Deny access to metadata files and directory listings
    Use short expiry times for session tokens
    Continuously test access control checks and log any failures

Cryptographic Failures

Cryptographic failures compromise the CIA Triad: Confidentiality, Integrity and Availability, which
are fundamental for cybersecurity, ensuring that data is accessed only by those authorised, that data
remains unmodified during transmission and that data and systems are available when needed by authorised users.
Here is a list of mitigation strategies:

    Use the strongest and most modern algorithms for hashing and encryption
    Encrypt data at rest and encrypt data in transit
    Use proper key management
    Do not store sensitive data if it is not needed

Injection

Here is a list of mitigation strategies:

    Use parameterised queries to define their structure first before passing user input
    Use filtering to reject bad inputs
    Use a Web Application Firewall to detect and block some types of injection

Insecure Design

Proper planning and threat modelling is essential to protect sensitive data and
prevent systemic vulnerabilities. Here is a list of mitigation strategies:

    Plan threat modelling in the design stage
    Use a defense in depth approach
    Define security requirements early

Security Misconfiguration

Here is a list of mitigation strategies:

    Reduce the attack surface by disabling anything unnecessary
    Do not use default credentials
    Implement secure HTTP headers
    Do not use detailed error messages
